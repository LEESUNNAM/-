# 바이브이선남 루트 프로젝트 가이드

## 폴더 역할

- 이 폴더(`E:\바이브이선남`)는 이선남님의 전체 프로젝트를 관리하는 루트 폴더입니다.
- 하위에 `lecture1` 등 개별 프로젝트 폴더가 위치합니다. 각 하위 프로젝트는 자체 `CLAUDE.md`가 있으면 그 규칙을 우선 따르고, 이 문서는 루트 공통 규칙입니다.
- 이 폴더 전체는 GitHub 저장소 **https://github.com/LEESUNNAM/-** (origin) 에 백업됩니다.

## 백업 규칙 — 매 작업 종료 시 자동 업로드

**중요: 어떤 작업(코드 수정, 파일 생성/삭제, 설정 변경 등)이든 끝나면, 사용자가 별도로 요청하지 않아도 변경사항을 곧바로 GitHub 저장소에 업로드한다.**

업로드 절차:

```bash
git add -A
git commit -m "{작업 내용을 한국어로 간단히 요약}"
git push
```

- 저장소 관리(생성/설정 확인 등)가 필요하면 `/gh-cli` 스킬을 활용한다 (`gh-cli/references/repo-admin.md` 참고).
- 커밋 메시지는 무엇을 했는지 한국어로 간단명료하게 작성한다.
- `.gitignore`에 명시된 파일(`node_modules`, `.env`, 빌드 산출물, Claude Code 로컬 설정 등)은 업로드 대상에서 제외한다.

## 금지 / 주의 사항

- `git push --force`, `git reset --hard`, 브랜치 삭제 등 되돌리기 어려운 작업은 사용자 확인 없이 실행하지 않는다.
- API 키, 토큰 등 민감 정보는 절대 커밋하지 않는다 — `.env` 파일과 `.gitignore` 처리를 항상 확인한다.
- 저장소(`gh repo delete` 등) 삭제나 강제 변경처럼 되돌리기 어려운 GitHub 작업은 `/gh-cli` 스킬의 안전 규칙을 따라 항상 먼저 확인받는다.
